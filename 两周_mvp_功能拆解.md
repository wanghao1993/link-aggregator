# 🚀 两周 MVP 功能拆解（社区型链接聚合）

> 目标：**两周内上线一个可浏览、可分享、可收藏的社区型链接聚合产品**

---

## 🎯 MVP 成功标准

上线时满足以下 5 条，即视为 MVP 成功：

- ✅ 未登录用户可浏览首页与合集页
- ✅ 合集页 URL 可直接分享访问
- ✅ 登录用户可收藏网站
- ✅ 登录用户可创建至少 1 个合集（Collection）
- ✅ 页面视觉与结构“值得被转发”

> **不在成功标准内的功能，一律不做。**

---

## 🗓 Week 1：内容结构与展示

### Day 1｜项目初始化

**目标：项目可运行、可部署**

- Next.js（App Router）
- Tailwind CSS + shadcn/ui
- Prisma + PostgreSQL
- 基础 Layout（Header / Container）

**产出**
- 首页 `/` 可访问
- 公共 Header 完成

---

### Day 2｜数据库结构（MVP 极简）

**必须表结构**

- User
- Collection
- Link
- CollectionLink（中间表）

**明确不做**
- 评论表
- 点赞表（先用收藏数）
- 通知系统

> 原则：Schema 宁少勿多，避免“未来设计”。

---

### Day 3｜合集页（最核心页面）

**目标：只做展示，不管登录**

- 路由：`/c/[slug]`
- 合集 Header（标题 / 简介 / 作者）
- Link 卡片列表
- 使用 SSG / ISR

**产出**
- 一个可被分享的合集页面

---

### Day 4｜首页（合集的集合）

- 精选合集列表
- 最近更新合集

> 首页本质是「合集卡片墙」，不做复杂 Feed。

---

### Day 5｜网站信息抓取（基础版）

- 服务端 fetch URL
- 自动解析：
  - title
  - description
  - favicon
- 解析失败时允许用户手动填写

**不做**
- 复杂爬虫
- JS 渲染站点解析

---

### Day 6｜创建合集（登录后）

- 新建合集：
  - 标题
  - 简介
  - 是否公开（默认公开）

- UI 形式：Modal

---

### Day 7｜向合集添加链接

- 粘贴 URL
- 自动解析网站信息
- 保存至指定合集

🎉 **Week 1 结束：产品已具备完整内容结构**

---

## 🗓 Week 2：用户行为与传播能力

### Day 8｜登录与权限（最小可用）

- 登录方式建议：
  - Email Magic Link
  - 或 GitHub OAuth

**权限规则**
- 未登录：只能浏览
- 已登录：可创建 / 收藏

---

### Day 9｜我的页面（个人中心）

- 路由：`/me`
- 我创建的合集
- 我收藏的链接

**不做**
- 拖拽排序
- 批量管理

---

### Day 10｜收藏功能（极简）

- 收藏 Link
- 收藏 Collection
- 仅做计数累加

> 不做推荐算法、不做 Feed 排序。

---

### Day 11｜分享与 SEO（增长关键）

- 合集页 SEO：
  - title
  - description
  - og:image（模板图即可）

- 支持一键复制链接

---

### Day 12｜链接状态标记（核心差异点）

为 Link 增加状态字段：

- `used`（我用过）
- `later`（待研究）

UI 展示：
- ✅ 我用过
- ⏳ 待研究

> 这是区别于普通书签的关键设计。

---

### Day 13｜关键页面视觉打磨

**只优化以下页面**

- 首页
- 合集页
- Link 卡片

**明确不碰**
- 表单细节
- 动效

---

### Day 14｜上线与种子内容

- 正式部署
- 手动创建：
  - 5–10 个高质量合集
  - 每个合集 ≥10 个链接

🚀 **MVP 正式上线**

---

## ❌ MVP 阶段禁止事项

以下任一都会严重拖慢进度：

- ❌ 原生 App
- ❌ 评论系统
- ❌ 算法推荐
- ❌ 积分 / 等级体系
- ❌ 多语言支持

---

## 🧠 MVP 心法

> **MVP 不是最小功能集合，**  
> **而是最小“值得被分享”的体验。**

你的产品，首先是给「点开链接的人」看的。

---

## 📌 下一步可扩展方向（非 MVP）

- Chrome 插件 / 系统分享入口
- 关注策展人 Feed
- AI 自动总结与标签
- 私有 / 团队合集

> 所有扩展，必须基于已有真实用户行为。
